# IR-2026-Grupo-7-ControlMotorAcceso
OpenRehab ACV es una plataforma de tests de control motor y hemiparesia destinado a pacientes de rehabilitación que sufrieron un ACV.

En primera instancia, se abrirá la ventana del inicio de sesión. El paciente deberá ingresar sus credenciales y en caso de que no se haya creado una cuenta, tiene la opción de registrar un nuevo usuario. Cuenta con varias alertas como elegir un nombre de usuario ya tomado o una contraseña menor a 8 caracteres, entre otras. Además, se da la opción de ocultar la contraseña. Luego, ingresa al menú.
El menú del proyecto cuenta con un mensaje de bienvenida personalizado según las credenciales completadas. Se optó por un diseño conciso, manteniendo el contraste para mayor claridad, evitando posibles distractores, pero sin dejar de lado la estética clínica. Si se deja el cursor encima de cada botón, se presenta un cartel que explica brevemente en qué consiste dicho test y qué valores medirá. 

ESTABILIZADOR DE TRAYECTORIA: El desafío es transitar un sendero laberíntico. La ventana muestra un contador en tiempo real desde que el paciente lleva el cursor a la zona de salida. Expone feedback inmediato, cambiando el cursor verde a rojo y de mayor tamaño cuando sale del camino, además de mostrar el timer en color rojo. Mide estadísticas en tiempo real, llevando un conteo de los errores (las veces que salió del camino) y revelando un timer que cronometra el tiempo que se encuentre fuera del sendero. A lo largo del camino se ubican objetivos en coordenadas aleatorias, los cuales son círculos blancos para asegurar contraste y visibilidad. Estos deben ser tocados para poder terminar el test, mostrando arriba de la pantalla un contador de los objetivos ya tocados. Si el paciente intenta llegar a la meta con objetivos aún activos, se resaltarán en rojo. Caso contrario, se mostrará el panel de métricas. El test evalúa cantidad de errores, tiempo fuera del camino, precisión (surge del cociente entre el tiempo fuera y dentro). Mide temblor, analiza las variaciones del ángulo en la posición del mouse y cuenta la cantidad de veces que el cambio fue significativo, considerando luego el tiempo final para expresar el valor en Hz. Por último, la rigidez es medida en rad/s según variaciones grandes en el ángulo de trayectoria que fueron realizados en grandes distancias. Además de las opciones de reintentar o salir, muestra las métricas récord (para fomentar consistencia) teniendo errores la mayor ponderancia y temblor o rigidez menor al ser fenómenos sobre los que el paciente no tiene mucho control.

ARRASTRE SOSTENIDO: El objetivo del test consiste en que el paciente empleando una tableta grafica, pinche con el cursor sobre el objeto rectangular, ejerciendo una presión superior a una presión umbral seteada. Luego debe arrastrar el objeto hasta la meta (un área color verde) sin dejar de emplear la presión superior a la umbral, de lo contrario el objeto volverá a su posición original. Cada vez que esto suceda se contara un error y simultáneamente se cronometrara el tiempo de finalización. Ademas el paciente dispondrá de una barra en la interfaz del juego donde podrá ver la fuerza que aplica sobre la tableta con el lápiz para poder concentrarse mejor en superar la mínima requerida. 

GANANCIA ADAPTATIVA: Este módulo evalúa y adapta en tiempo real la sensibilidad del mouse para pacientes con hemiparesia post-ACV que tienen rango de movimiento (ROM) limitado en miembro superior. El test funciona en tres fases: primero calibra el ROM real del paciente pidiéndole que mueva el mouse libremente durante 5 segundos; después presenta 15 trials donde tiene que alcanzar y clickear círculos que aparecen en distintas posiciones de la pantalla; y al finalizar exporta automáticamente un JSON con todos los datos en la carpeta /results/.
La ganancia se ajusta dinámicamente: si el paciente tiene un ROM muy chico, el cursor se amplifica (gain alto) para que pequeños movimientos lleguen lejos. A medida que encadena aciertos, la ganancia baja progresivamente para mantener un desafío motor adecuado. Si falla por timeout, la ganancia sube para facilitarle la tarea.
Qué mide: tiempo de reacción por trial, trayectoria completa del cursor (coordenadas reales y virtuales en el tiempo), cantidad de clicks errados, ganancia usada en cada trial, eficiencia del movimiento (distancia directa al target / distancia total recorrida), ROM calibrado, y comparación de gain inicial vs final como indicador de mejora intrasesión.

Instrucciones para instalar dependencias y luego correr el programa principal desde la raíz del repositorio:
```
pip install -r requirements.txt
python ing_rehab_grupo7/main.py
```
